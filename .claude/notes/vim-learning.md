# Обучение Vim/Neovim: от чайника до уверенного пользователя

Этот файл — ПРО ДРУГОЕ, чем `progress.md`. `progress.md` — техническая история
настройки конфига (плагины, LSP, форматирование и т.п.). **Этот файл** — про
личный прогресс пользователя в изучении САМОГО Vim: режимы, motions, операторы,
команды редактирования. Конфиг уже даёт IDE-обвязку (LSP/telescope/neo-tree и
т.п.), но база самого Vim — с нуля ("полный чайник" на момент старта, 2026-07-13).

Идём так же, как с конфигом: атомарно, шаг за шагом, объясняем "зачем" перед
"как", закрепляем практикой на реальных файлах (см. [[testing-prefer-tmux]] —
то же правило применимо и здесь: реальная практика в tmux/реальном nvim, не
голословные объяснения).

Параллельно ведётся живая шпаргалка по уже пройденным командам —
`vim-cheatsheet.md` в этой же папке. Пополнять её сразу после каждого урока.

## Формат отслеживания
- Каждый этап = один навык или тесно связанная группа команд.
- Отмечаем ✅ когда пользователь реально попробовал и уверенно воспроизводит
  (не когда просто объяснили).
- Порядок гибкий — можно менять местами по запросу пользователя.

## Роадмап

1. **Режимы и базовое перемещение**
   - Normal / Insert / Visual / Command-line, `Esc` — центральная идея Vim
   - `h j k l` (принципиально не стрелки — приучаем руки)
   - слова: `w` `b` `e` `ge`
   - строка: `0` `^` `$`
   - файл: `gg` `G`, `{count}G` / `:{N}`
   - экран: `H` `M` `L`, `Ctrl-d` `Ctrl-u` (полу-скролл)
   - посимвольный поиск в строке: `f` `F` `t` `T` `;` `,`

2. **Базовое редактирование**
   - вход в Insert: `i` `a` `I` `A` `o` `O` — разница между ними
   - `x`, `dd`, `yy`, `p` `P`, `u`, `Ctrl-r` (redo)
   - счётчики перед командой: `3dd`, `5j`, `10x`

3. **Операторы + motions (главная суперсила Vim)**
   - идея "оператор + motion" = комбинаторика: `d`/`c`/`y` + любое движение
     (`dw`, `d$`, `dG`, `cw`, `y0`...)
   - dot-repeat: `.`
   - текстовые объекты: `iw`/`aw`, `i"`/`a"`, `ip`/`ap`, `it`/`at`, `i(`/`a(`
     и т.д. — "inner" vs "around"

4. **Поиск и замена**
   - `/` `?` поиск, `n` `N` навигация по совпадениям
   - `:s/.../.../` с флагами (`g`, `c`), `:%s` на весь файл
   - `*` `#` — поиск слова под курсором

5. **Визуальные режимы**
   - `v` (character), `V` (line), `Ctrl-v` (block)
   - операции над выделением (`d`, `y`, `c`, `>`/`<` для отступов)

6. **Регистры и буфер обмена**
   - именованные регистры `"a`-`"z`
   - `"+` — системный clipboard (уже неявно юзается конфигом?)
   - `"0` — yank register (переживает `d`, в отличие от unnamed)

7. **Метки и макросы**
   - marks: `m{буква}`, переход `` `{буква} ``
   - макросы: `qa` ... `q` (запись), `@a` (проигрывание), `@@` (повтор),
     `{count}@a`

8. **Нативная многофайловая работа: буферы/окна/вкладки**
   - `:bnext`/`:bprev`, `:split`/`:vsplit`, `Ctrl-w` + `hjkl` навигация
   - сравнить с тем, что уже даёт конфиг через плагины (bufferline,
     `<C-hjkl>` — см. `progress.md`) — где нативная база, где плагин лишь
     удобная обёртка

9. **Command-line и полезные ex-команды**
   - диапазоны (`:1,5d`, `:.,+3d`)
   - `:g/pattern/d` (global command) — мощная штука для батч-правок
   - `:norm` — применить normal-команду к диапазону строк

10. **Продвинутое (по желанию, после уверенности в 1-9)**
    - jumplist: `Ctrl-o` / `Ctrl-i`
    - changelist: `g;` / `g,`
    - folds (в конфиге уже есть базовая настройка `foldlevel=99` — разобрать,
      как ими пользоваться руками: `za`, `zo`, `zc`, `zR`, `zM`)

## Открытые вопросы / на будущее
- Как соотносится изученное с уже настроенными в конфиге кеймапами
  (flash.nvim `s`/`S` вместо нативного `f`/`t` для дальних прыжков,
  LazyVim-раскладка `<leader>...` и т.п.) — стоит явно проговаривать разницу
  "нативный Vim" vs "то, что добавил конфиг", чтобы не путать источники.
