# Vim/Neovim: шпаргалка по пройденным командам

Живой справочник команд из [[vim-learning]] — только то, что уже разобрано
(необязательно ✅-подтверждено практикой, но уже объяснено). Пополняется по
мере уроков, в том же порядке, что и роадмап там.

Отдельным блоком в конце — команды от ПЛАГИНОВ (не нативный Vim), которые
заменяют/расширяют штатное поведение. Помечены явно, чтобы не путать источник
(см. "Открытые вопросы" в vim-learning.md).

## 1. Режимы и базовое перемещение

**Режимы**
| Клавиша | Действие |
|---|---|
| `Esc` | вернуться в Normal (базовый режим) |
| `i` / `a` / `I` / `A` / `o` / `O` | войти в Insert (см. урок 2) |
| `v` / `V` / `Ctrl-v` | войти в Visual (см. урок 5) |
| `:` | Command-line |

**По символам**
| Клавиша | Действие |
|---|---|
| `h` | влево |
| `l` | вправо |
| `j` | вниз |
| `k` | вверх |

**По словам**
| Клавиша | Действие |
|---|---|
| `w` | к началу следующего слова |
| `b` | к началу предыдущего слова |
| `e` | к концу текущего/следующего слова |
| `ge` | к концу предыдущего слова |

**По строке**
| Клавиша | Действие |
|---|---|
| `0` | в начало строки (с пробелами) |
| `^` | к первому непробельному символу |
| `$` | в конец строки |

**По файлу**
| Клавиша | Действие |
|---|---|
| `gg` | в начало файла |
| `G` | в конец файла |
| `{N}G` или `:{N}` | на строку N |

**По экрану**
| Клавиша | Действие |
|---|---|
| `H` | верх видимого экрана |
| `M` | середина экрана |
| `L` | низ экрана |
| `Ctrl-d` | прокрутка вниз на полэкрана |
| `Ctrl-u` | прокрутка вверх на полэкрана |

**Посимвольный поиск в строке**
| Клавиша | Действие |
|---|---|
| `f{c}` | вперёд до символа `c` (на него) |
| `F{c}` | назад до символа `c` (на него) |
| `t{c}` | вперёд до символа `c` (перед ним) |
| `T{c}` | назад до символа `c` (после него) |
| `;` | повторить последний f/F/t/T |
| `,` | повторить в обратную сторону |

## 2. Оператор `c` (change)

Общая идея операторов (см. [[vim-learning]] шаг 3): `c` + motion/text-object —
удаляет то, что накрывает motion, и сразу переводит в Insert (в отличие от
`d`, после которого остаёшься в Normal).

| Команда | Действие |
|---|---|
| `c{motion}` | заменить текст, накрытый motion (`c$`, `c0`, `cG`...) |
| `cc` | заменить всю строку, сохранив отступ (алиас `S`) |
| `c$` | заменить от курсора до конца строки (алиас `C`) |
| `cw` | заменить слово от курсора (см. ВАЖНО ниже) |
| `ce` | заменить до конца слова |
| `cf{c}` / `ct{c}` | заменить вперёд до символа `c` (включительно/перед ним) |
| `ciw` / `caw` | заменить слово целиком (inner/around, text object) |
| `ci"` / `ca"` | заменить содержимое кавычек (без/с самими кавычками) |
| `ci(` / `ci{` / `ci[` | заменить содержимое скобок |
| `cit` / `cat` | заменить содержимое HTML/XML-тега (без/с тегами) |
| `.` | повторить последнюю команду изменения (включая `c...`) |

**ВАЖНО про `cw`**: исторический сюрприз Vim — `cw` ведёт себя как `ce`
(меняет только слово, не трогая пробел после него), хотя по логике "оператор
+ motion" ожидался бы результат как у `dw` (которая съедает и пробел до
следующего слова). Это единственное такое специальное исключение — не путать
с общим правилом "оператор+motion = комбинаторика".

## 3. Текстовые объекты (text objects)

Работают ТОЛЬКО в двух контекстах: после оператора (`d`/`c`/`y`/`gs a`/`gs d`
и т.п. — operator-pending) или в Visual (`v`/`V`/`Ctrl-v`). Сами по себе в
Normal не существуют — `iw` там начнётся с `i` = вход в Insert, а не текстовый
объект.

Синтаксис: `{i|a}{объект}` — `i` (inner) только содержимое, `a` (around) —
содержимое + окружение (пробел/сами кавычки-скобки/сам тег).

| Объект | Inner | Around |
|---|---|---|
| слово | `iw` | `aw` (+ пробел после) |
| WORD (по пробелам, без учёта пунктуации) | `iW` | `aW` |
| предложение | `is` | `as` |
| абзац | `ip` | `ap` |
| `"..."` / `'...'` / `` `...` `` | `i"` `i'` `` i` `` | `a"` `a'` `` a` `` |
| `(...)` | `i(` (алиас `ib`) | `a(` (алиас `ab`) |
| `{...}` | `i{` (алиас `iB`) | `a{` (алиас `aB`) |
| `[...]` | `i[` | `a[` |
| HTML/XML тег `<div>...</div>` | `it` | `at` |

Примеры:
- `diw` / `daw` — удалить слово (без/с пробелом после)
- `ci"` — заменить содержимое кавычек, оставив сами кавычки
- `yi(` — скопировать содержимое скобок
- `vi(` затем ещё раз `i(` — расширить визуальное выделение на внешние скобки
  (удобный способ "расти" наружу по вложенности)

## Плагины

### mini.surround — обёртки (кавычки/скобки/теги)

Не нативный Vim — плагин `nvim-mini/mini.surround` (`lua/plugins/mini-surround.lua`).
Все команды на `gs`-префиксе (в LazyVim/апстриме по умолчанию `gs` — сохранили).

| Клавиша | Действие |
|---|---|
| `gsa{motion/textobj}{c}` | добавить обёртку `c` вокруг motion/text object |
| `gsa` (в Visual, после выделения) | добавить обёртку `c` вокруг выделения |
| `gsd{c}` | удалить обёртку `c` (курсор внутри/на ней) |
| `gsr{old}{new}` | заменить обёртку `old` на `new` |
| `gsf{c}` | прыгнуть к обёртке `c` справа от курсора |
| `gsF{c}` | прыгнуть к обёртке `c` слева от курсора |
| `gsh{c}` | подсветить обёртку `c` (визуально проверить, что попадёт под действие) |
| `gsn{N}` | изменить радиус поиска обёртки в строках (`n_lines`) |

Идентификаторы обёртки `c`: обычный символ (`"`, `'`, `` ` ``, `(`, `)`, `[`,
`]`, `{`, `}`) — **открывающая** скобка добавляет с отступом-пробелом
(`( x )`), **закрывающая** — без (`(x)`); плюс спецсимволы `t` (тег, спросит
имя) и `f` (вызов функции, спросит имя).

Примеры:
- `gsaiw"` — обернуть слово под курсором в `"..."`
- `gsaiw)` — обернуть слово в `(word)` (без пробелов)
- `gsaiw(` — обернуть слово в `( word )` (с пробелами)
- `gsd"` — снять кавычки вокруг курсора
- `gsr"'` — заменить `"..."` на `'...'`
- `viw` затем `gsa"` — обернуть выделение в кавычки
- `gsat` затем ввести имя тега (`div<CR>`) — обернуть в `<div>...</div>`
